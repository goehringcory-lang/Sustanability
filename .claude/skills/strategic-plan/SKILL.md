# Sustain — Strategic Working Plan

This skill wraps the strategic plan document for the Sustain sustainability website. The full plan lives at `Sustain_Strategic_Plan.md` in this skill directory.

## When to use this skill

Pull the plan into context whenever the user's request touches:

- **Monetization** — affiliate programs, commission rates, product margins, pricing
- **Content strategy** — what to publish, how often, in which pillar (Catalog / Journal / Notebook)
- **Brand voice** — proposed copy, taglines, headlines, button text (check §4.3 guardrails)
- **Product** — anything about The Canister, manufacturing, packaging, Kickstarter
- **SEO & growth** — keyword clusters, backlinks, email capture, the lead magnet
- **Roadmap & sequencing** — what to build now vs later, the 90-day action plan
- **Risks & tradeoffs** — when a change might violate the "do not do" list (§12)

## How to use it

1. **Read the relevant section first** — `Sustain_Strategic_Plan.md` is in this directory. Don't answer strategy questions from memory of past conversations.
2. **Quote the plan when relevant** — cite the section number (e.g. "per §4.3 guardrail #6") so the user can verify.
3. **Flag conflicts loudly** — if the user asks to do something the plan explicitly counsels against (add Instagram in Phase 1, accept sponsored content, launch multiple SKUs at once, write listicle content for Google), stop and surface the conflict before implementing. Don't just go along.
4. **Treat the voice guardrails as code rules** — guardrails in §4.3 (no em dashes, no banned words, downside-first reviews, etc.) apply to any copy you write or edit on the site.

## Banned words (do not introduce in any site copy)

From §4.3 guardrail #6: *eco-friendly, game-changer, must-have, hack, dupe, curated (as a verb), sustainable (as a modifier when a more specific word exists)*. Also: no em dashes — use commas, colons, or parentheses.
